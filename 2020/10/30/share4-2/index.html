浏览器提供了我们开发工具,我们就要好好利用,多了解一下他的功能

按下F12

Elements

这里存放着html源代码和css样式
对某一个html元素右键,可以对对元素的属性进行修改.也可以直接对整个网页元素进行修改(Edit as HTML)
copy可以复制当前元素的许多信息,可以复制整个元素,JS的选择路径,css…可以快速将页面移动到所选元素处,最重要的是他可以打断点,所有的断点会被写在一旁的DOM Breakpoints里,这一行的Event Listener页面，可以查看到该元素的所有监听事件。在开发中，尤其是维护其他人的代码时，会出现不了解元素对应的监听事件，这个时候，可以在这个页面中找到。这个页面不仅能看到对应的事件函数，还可以定位该函数所在的JS文件以及在该文件中的具体位置（行数），大大提高开发维护的效率。Properties 中还提示我们 $0指向我们现在选中的元素,我们可以通过console.dir($0)来访问这个元素的所有属性.

元素被打断点之后如果被js代码改变,页面会停止加载,方便我们查看元素属性
还可以直接将元素作为全局变量进行输出, 非常便于调试

Console

控制台的主要工作还是输出js内部的细节以及报错等等,对于不确定的输出,也可以在这里进行尝试.
$() | 返回与指定的CSS选择器相匹配的第一个元素，等同于document.querySelector()
$$() | 返回与指定的CSS选择器相匹配的所有元素的数组，等同于document.querySelectorAll()

$x() | 返回与指定的XPath相匹配的所有元素的数组

Sourcs

再来简单的说一下断点, 断点就是开发工具提供的一种功能,可以让程序在我们指定的地方中断运行, 我们在这个时候去观察某些变量的值,函数的堆栈,这样可以让我们更加清楚在程序内部数据是如何变化流动的.这比我们反复修改程序在里面增删console.log高效多了

例如对上面这个js打断点 在return b(argu + "2")打点,就可以看到argu的值是”1”
还可以对断点添加条件,当符合条件时,浏览器中断运行.
一侧的辅助框还可以在断点间跳动,继续运行…
call stack记录了当前行的函数调用栈,可以清楚地看见函数的运行.

Network

从红点开始往右
记录按钮 处于打开状态时会在此面板进行网络连接的信息记录，关闭后则不会记录。
清除按钮 清除当前的网络连接记录信息

还可以对所有的请求进行相应的筛选,让我们不用在长长的列表中来回翻找.

一些条件

  • domain：资源所在的域(url中的域名部分)
  • has-response-header：资源是否存在响应头
  • is：当前时间点在执行的请求。
  • larger-than：显示大于指定值大小规格的资源。单位是字节(B)
  • method：使用何种HTTP请求方式。如 GET
  • mime-type：也写作content-type，是资源类型的标识符。如 text/html
  • scheme：协议规定。如 HTTPS
  • set-cookie-name：服务器设置的cookies名称
  • status-code：HTTP响应头的状态码

能够根据时间，查看对应时间下 浏览器请求的资源信息

Preserve log：是否保留日志

当选择保留日志，重新加载url当前界面时，之前请求显示的资源信息，会保留下来

Disable cache：是否进行缓存

当打开开发者工具时生效，打开这个开关，则页面资源不会存入缓存，可以从Status栏的状态码看文件请求状态。

offline : 设置模拟速度,3G, 4G, WiFi…
对着请求右键也可以对相应的信息进行复制.

最上面的长条记录了各个请求收到回复的时间和处理文件的时间长短

performance

这个选项可以记录我们的浏览动作,比如上下滚动,交互等等,与此同时,他还记录下了关于CPU的信息,可以看到FPS和NET,也就是说在这我们可以看到那些时候哪些动作是最耗费资源的,这样我们就可以对相应的部分进行优化.如果在css动画和布局的计算上面消耗了大量资源,那么对js部分的数据计算就会有影响,在进行布局是要对样式的应用有选择.

1、大开销样式计算影响响应或动画
　　任何会更改元素几何形状的 CSS 属性，如宽度、高度或位置；浏览器必须检查所有其他元素并重做布局。避免会触发重排的CSS属性

　　2、复杂的选择器影响响应或动画
嵌套选择器强制浏览器了解与所有其他元素有关的全部内容，包括父级和子级。尽量在CSS中引用只有一个类的元素

由于手机cpu运算速率没有电脑高,我们也可以用电脑浏览器对手机的计算负载进行模拟

Application

该面板主要是记录网站加载的所有资源信息，包括存储数据（Local Storage、Session Storage、IndexedDB、Web SQL、Cookies）、缓存数据、字体、图片、脚本、样式表等。

Local Storage 如果你在开发过程中使用了local storage来存储键值对(KVPs)，那么你就可以通过Local Storage窗格来检查、新增、修改、删除这个键值对。
Application Cache 你可以使用Application Cache窗格去查看通过Application Cache API创建的资源。
